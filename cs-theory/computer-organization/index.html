存储系统

DRAM、SRAM、ROM

DRAM

使用栅极电容存储信息。

  • 一般以 2ms 周期刷新。
  • 破坏性读出，需要重写。
  • 使用行列地址减少选通线数量。
  • 使用行列地址分两次送的方式减少地址线数量。

SRAM

使用双稳态触发器存储信息。

无需重写和刷新，元件复杂导致集成度较低。

ROM

MROM -> PROM -> EPROM -> EEPROM -> Flash Memory -> SSD

BIOS: 自举装入程序。

主存储器

BIOS ROM + RAM

主存与 CPU 连接

主存器通过数据总线，地址总线和控制总线与 CPU 连接。地址总线的位数决定了可寻址的最大内存空间。控制总线指出总线周期类型和本次输入输出操作完成的时刻。

位扩展法

每个存储器的数据线都并行连接在CPU的数据总线上，而地址位数是串行相连。

字扩展法

通过译码器，将多个字数相同的存储芯片并联起来，增加位数，并且存储空间是连续的。

字扩展方法只能增加主存储器的容量，存取速度没有提升。

双端口 RAM

支持两个 CPU 同时访问取数据。

高位交叉编址（$nT$）和低位交叉编址（$T + (n - 1)r$），编组的模块数 $m \ge T/r$。

Cache

在现在的系统中，处理器需要上百个时钟周期才能从主存中取到数据。如果没有cache，处理器在等待数据的大部分时间内将会停滞不动。

时间局部性原理和空间局部性原理

如果一个数据现在被访问了，那么以后很有可能也会被访问。如果一个数据现在被访问了，那么它周围的数据在以后可能也会被访问。

映射方式

  1. 全相联映射——主存块可以放在 Cache 的任意位置。
  2. 直接映射——每个主存块只能放在一个特定的位置。Cache块号 = 主存块号 % Cache块总数。
  3. 组相联映射——Cache块分为若干组，每个主存块可以放到特定分组中的任意一个位置。组号 = 主存块号 % 分组数

替换算法

  1. 随机算法(RAND)
  2. 先进先出算法(FIFO)
  3. 近期最少使用(LRU)

写策略

命中时：

  1. 回写法：当CPU对Cache写命中时，只修改Cache的内容，而不立即写入主存，只有当次块被换出时才写回主存。减少了访存次数，但存在数据不一致的隐患。
  2. 全写法：当CPU对Cache写命中时，必须把数据同时写入Cache和主存，一般使用写缓冲。访存次数增加，速度变慢，但是能保证数据的一致性。无脏位。

不命中时：

  1. 写分配法——当CPU对Cache不命中时，把主存中的块调入Cache，在Cache中修改。通常搭配回写法使用，改完后要被换出，才在主存中修改。
  2. 非写分配法——当CPU对Cache写不命中时，只写入主存，不调入Cache，搭配全写法使用。
